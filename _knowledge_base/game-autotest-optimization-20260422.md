# SanGuoPai 遊戲自動測試與參數調優紀錄

日期：2026-04-22
狀態：已完成並推送至 GitHub

---

## 1. 背景與目標

### 1.1 問題陳述

SanGuoPai（三國派）是一款結合 21 點機制的 roguelike 卡牌遊戲。在開發過程中發現幾個核心問題：

1. **技巧深度不足**：純 21 點 + 基礎武將技能，玩家主要靠運氣獲勝
2. **難度曲線不合理**：三種難度（入門/標準/困難）之間區分度不夠
3. **缺乏數據支撐**：平衡調整憑感覺，無法量化不同水平玩家的體驗差異
4. **通關後動力不足**：普通難度通關後，困難模式沒有足夠的機制深度支撐長期遊玩

### 1.2 目標

建立一個**無人值守、自我迭代**的自動測試框架，同時解決：

- **策略優化**：找到最接近最優的 AI 策略，驗證遊戲是否有足夠技巧空間
- **參數調優**：確保三種難度對應三種玩家水平都有合理的挑戰曲線
- **機制驗證**：用數據證明新機制（棄牌、分牌、武將組合）確實提升了技巧深度

---

## 2. 方法論：雙框架設計

受 Karpathy [autoresearch](https://github.com/karpathy/autoresearch) 啟發，設計了兩個獨立但互補的框架：

### 2.1 Agent 框架 — 策略優化

| 組件 | 角色 | 是否可變 |
|------|------|---------|
| `agent.js` | **唯一可變基因組** — 決策邏輯 | **可編輯** |
| `harness.js` | 固定測試框架 — 運行遊戲、收集指標 | 不可編輯 |
| `program.md` | 研究策略文檔 | 人類設定 |

**Fitness 公式**：
```
fitness = Σ(stage_cleared × difficulty_multiplier) / total_runs

難度乘數：入門=1.0, 標準=2.0, 困難=3.5
```

### 2.2 Tuner 框架 — 參數調優

模擬三種玩家類型（新手/休閒/硬核），測量參數改動對各類玩家的影響：

**Fitness 公式**：
```
fitness = balance_score × 0.7 + skill_expression_bonus × 0.3

balance_score = 各難度×各玩家類型的目標達成率平均值
skill_expression = hardcore_avg_stages - newbie_avg_stages（技巧表達度）
```

### 2.3 為什麼需要兩個框架？

- **Agent** 回答：「給定當前參數，最強策略能走多遠？」
- **Tuner** 回答：「給定當前機制，不同水平的玩家體驗如何？」
- 兩者結合才能確保：遊戲**既有深度**（Agent 能持續優化）**又有良好曲線**（Tuner 確保各水平玩家都有挑戰）

## 3. 迭代過程：失敗 → 分析 → 修正

### 3.1 第一輪：公式問題暴露（Exp 001-022）

**初始狀態**：
- 傷害公式：`p * (31 - p)`（點數接近 21 時傷害最高，但 21 點和 20 點差距不大）
- 武將池：28 名，技能類型單一（只有加法/乘法）
- 爆牌懲罰：完全沒有金錢

**關鍵發現（Exp 016-021）**：

| 實驗 | Fitness | Balance | Skill Expression | 問題 |
|------|---------|---------|-----------------|------|
| 016 | 0.316 | 0.451 | **-1.87** | 硬核玩家表現極差 |
| 021 | 0.584 | 0.834 | **-0.25** | 硬核 < 新手（入門難度）|

**診斷**：`p*(31-p)` 公式下，保守策略（低點數停牌）幾乎總是最優的。因為：
- 要牌風險高（爆牌 = 0 收益）
- 武將技能加成不足以彌補要牌風險
- 因此「懂遊戲機制的硬核玩家」反而因為過度激進而輸給「保守的新手」

**結論**：公式設計本身**壓制了技巧表達空間**。

### 3.2 第二輪：公式重構（Exp 023-038）

**核心修改**：

1. **傷害公式**：`p * (31 - p)` → `p * (p + 5)`
   - 原公式：21 點 = 210，20 點 = 220（20 點反而更高！）
   - 新公式：21 點 = 546，20 點 = 500，15 點 = 300
   - 新公式是**嚴格遞增**的，高點數始終更有價值

2. **爆牌保底獎勵**：爆牌不再 = 0
   ```js
   // 22 點保留 70%，23 點 60%，... 29+ 點 0%
   keepRatio = max(0, 0.8 - overage * 0.1) + bustProtection
   ```

3. **武將池擴展**：28 → 43 名，新增 6 種機制：
   - `bonus_hands`：額外出牌次數
   - `extra_draw`：初始多抽牌
   - `bust_protection`：爆牌保底加成
   - `ai_threshold_offset`：干擾敵人 AI
   - `hit_limit_bonus`：增加要牌上限
   - `enemy_bust_bonus`：敵人爆牌時額外獎勵

**效果對比**：

| 指標 | 修改前（Exp 021） | 修改後（Exp 038） | 變化 |
|------|------------------|------------------|------|
| Fitness | 0.584 | **0.855** | +46% |
| Balance Score | 0.834 | **0.847** | +1.6% |
| Skill Expression | **-0.25** | **+2.62** | +2.87 stages |
| 硬核玩家（入門）| 3.88 stages | **7.62 stages** | +96% |
| 硬核玩家（標準）| 3.24 stages | **5.24 stages** | +62% |

**關鍵洞察**：
- 新公式使「追求高點數」成為正期望值策略
- `bust_protection` 武將進一步降低了激進策略的風險
- 武將組合搭配（如 諸葛亮+張遼 = 不爆牌+0 要牌加成）創造了真正的構築深度

## 4. Agent 策略優化結果

| 實驗 | Fitness | Max Stages | 關鍵改進 |
|------|---------|-----------|---------|
| 001 | 11.68 | 6 | 基準策略（~15 點停牌） |
| 002 | 13.17 | 6 | 加入敵人明牌適應 |
| 004 | 31.44 | 9 | 加入武將聯動調整 |
| 006 | **35.81** | **10** | 完善分牌/雙倍/棄牌時機 |

**最終 Agent 策略特點**：
- 基準停牌閾值：15（數學最優）
- 根據武將動態調整：
  - 諸葛亮/張遼 → 閾值降至 13（追求不爆牌加成）
  - 關羽/趙雲 → 閾值升至 16（追求 20/21 加成）
- 敵人明牌適應：敵人亮 10 → 更保守；敵人亮 2-6 → 更激進
- 棄牌策略：弱牌（<10）或敵人強（亮 10）時果斷棄牌重抽

## 6. 核心經驗教訓

### 6.1 公式設計決定技巧天花板

傷害公式是整個遊戲技巧深度的**基礎設施**：

| 公式 | 特性 | 技巧空間 |
|------|------|---------|
| `p*(31-p)` | 凸函數，最佳點在 15-16 | 極低 — 保守策略幾乎總是最優 |
| `p*(p+5)` | 嚴格遞增 | 高 — 高點數始終更好，激進策略有回報 |

**經驗**：在設計任何數值系統時，先畫出收益曲線，確認它是否給「高風險高回報」策略留有空間。

### 6.2 單一標量指標的重要性

兩個框架都使用**單一 fitness 值**，這是 autoresearch 模式的關鍵：
- 沒有單一指標 → 每次改進都要人工判斷「這個好還是那個好」
- 單一指標 → 實驗可以自動排序、自動保留最優、自動回滾

### 6.3 負向 Skill Expression 是致命信號

當「硬核玩家表現不如新手」時，這不是玩家問題，是**遊戲機制問題**。此時不應該調整參數，而應該回到底層公式重構。

### 6.4 武將設計的多樣性比數量重要

從 28 → 43 名武將，但真正產生技巧深度的是**新機制類型**：
- `bust_protection` 改變了風險計算
- `hit_limit_bonus` 改變了要牌策略
- `extra_draw` 改變了初始手牌質量

單純增加「加法 20 vs 加法 25」的武將不會創造新的策略空間。

## 7. 後續擴展方向

1. **對抗訓練**：讓 Agent 同時控制敵人 AI，互相博弈提升
2. **遺傳算法交叉**：將多個高分 Agent 的決策規則組合
3. **在線 A/B 測試**：將 tuner 參數與真實玩家數據對齊
4. **新機制驗證**：每次加入新機制（如連擊系統、元素反應），先用 tuner 驗證是否提升 skill expression
