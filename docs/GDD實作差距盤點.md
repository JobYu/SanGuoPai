# 《三國派》GDD 實作差距盤點

更新日期：2026-03-31

## 目的

本文對照 [GDD.md](../GDD.md) 與目前專案實作，整理出：

- 已實作的核心內容
- 部分實作但尚未完整落地的內容
- 尚未實作的內容
- 與 GDD 已明顯偏離的方向

## 結論摘要

目前專案已具備可玩的核心戰鬥原型，包括：

- 21 點對戰主循環
- 要牌 / 出牌 / 棄牌
- 敵方暗牌與基本 AI
- 傷害結算與敵將氣勢
- 金錢獎勵
- 招募 / 出售 / 擴展槽位
- 基本錦囊與寶物

但距離 GDD 的完整版本，仍缺少長線成長、模式擴展、音效與平台落地等內容。

## 一、已實作

### 1. 核心戰鬥規則

已實作項目：

- 玩家與敵將進行 21 點對決
- 雙方初始各兩張牌，敵方一明一暗
- 玩家可要牌、出牌、棄牌
- 出牌時才判定爆牌
- 敵方依 AI 傾向決定是否繼續抽牌
- 勝利後對敵將氣勢造成累積傷害
- 出牌次數用盡且未擊破敵將時遊戲失敗

對應代碼：

- [js/game_ui.js](../js/game_ui.js)
- [js/engine/ai.js](../js/engine/ai.js)
- [js/engine/combat.js](../js/engine/combat.js)
- [js/engine/blackjack.js](../js/engine/blackjack.js)

### 2. 傷害與金錢原型

已實作項目：

- 傷害公式 `(基礎傷害 + 加法加成) × 乘法倍率`
- 勝利傷害會扣減敵將氣勢
- 造成的傷害同時轉為金錢
- 結算畫面會展示本次點數與加成結果

### 3. 招募與養成原型

已實作項目：

- 戰後獎勵頁
- 武將隨機招募
- 出售武將
- 武將槽位擴展
- 擁有金錢不足時的限制判定

### 4. 基本詳情彈窗

已實作項目：

- 點擊我方武將可查看插圖、稀有度、陣營、技能與 flavour
- 點擊敵將可查看插圖、AI 性格、停牌閾值與初始士氣

## 二、部分實作

### 1. 武將技能系統 ✅ 已完成

GDD 要求的觸發時機包含：

- 結算時
- 要牌時
- 停牌時
- 達成 21 點時
- 爆牌時
- 回合開始時

目前情況（2026-03-31 更新）：

- ✅ 已明確接入：`結算時`、`達成 21 點時`、`爆牌時`
- ✅ 已接入：`要牌時`（關平、甘寧）、`回合開始時`（諸葛亮）
- ⏸️ 預留但未使用：`停牌時`（目前無武將使用此觸發時機）

實作細節：

- **要牌時**：關平抽到 A 時標記，結算時 +75；甘寧首次要牌時標記，結算時 +90
- **回合開始時**：諸葛亮啟動觀星，若本回合未爆牌則結算時倍率 x2.2
- 技能效果通過狀態追蹤旗標（`guanPingActive`、`ganNingActive`、`zhugeLiangActive`）實現跨階段觸發

### 2. 陣營羈絆 ✅ 已完成

GDD 定義：

- 魏：控制與棄牌
- 蜀：錦囊與反擊
- 吳：連擊與花色
- 群：隨機與風險

目前情況（2026-03-31 更新）：

- ✅ 魏：3魏時額外 +1 棄牌次數
- ✅ 蜀：3蜀時錦囊折價 20% + **反擊機制**（輸掉時造成 30+(蜀將數-3)*10 傷害）
- ✅ 吳：3吳時三同花倍率 x1.1
- ✅ 群：3群時隨機加法傷害（50 ± 10）

結論：

- 四大陣營羈絆已完整實作

### 3. 錦囊與寶物

目前已接入：

- 錦囊：`借東風`、`空城計`
- 寶物：有資料與持有展示，至少部分效果接入

目前不足：

- 種類偏少
- 全局被動體系仍很薄
- 如「傳國玉璽提升稀有度」這類設計，尚未形成完整稀有度/掉落控制系統

### 4. 編年史關卡

目前情況：

- 敵將資料已有多個 stage 與 campaign 欄位
- 遊戲內流程會依 `stage_index` 依序推進

未完成部分：

- 沒有章節切換 UI
- 沒有戰役專屬規則或事件
- 沒有明確呈現「黃巾之亂 / 討伐董卓 / 群雄逐鹿 / 官渡 / 赤壁」的分段體驗

結論：

- 關卡資料已鋪好，但戰役表現仍是線性關卡串接

### 5. 詳情與圖鑑體驗

GDD 提到：

- 頂部點擊武將卡可查看詳情
- 詳細技能說明可在圖鑑或長按卡牌時展示

目前情況：

- 點擊查看詳情已做
- 長按沒有做
- 獨立圖鑑沒有做
- 歷史背景內容仍主要依賴 flavour 文案，未形成完整資料頁

## 四、與 GDD 偏離的地方

### 1. 技術方向已從 Electron 轉為 Web

GDD 仍寫：

- Electron + HTML/CSS/JavaScript
- PC / Steam 分發

但目前專案實際上已轉為：

- Vite Web 專案
- 使用 `fetch()` 載入資料
- 以網頁部署為主要路線

這不一定是問題，但表示 GDD 技術章節已過時，需要同步更新。

### 2. 視覺風格與 GDD 不一致

GDD 傾向：

- 古樸木質 / 竹簡質感

README 現況描述：

- Glassmorphism
- 現代 Web 視覺升級

這代表美術方向已經偏離原始設計稿，需要二選一：

- 以現在版本為準，回寫 GDD
- 或重新把產品視覺拉回原設計

### 3. Canvas 未採用

GDD 提到可用 DOM / Canvas 並考慮像素畫縮放。

目前實作為純 DOM 畫面與 CSS 縮放，沒有 Canvas 戰場層。
